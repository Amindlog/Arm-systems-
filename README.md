## Быстрый старт с Docker

Самый простой способ запустить систему - использовать Docker Compose.

### Требования

- Docker (v20+)
- Docker Compose (v2+)

### Запуск

1. **Клонируйте репозиторий и перейдите в директорию проекта:**

```bash
cd /Arm-systens-
```

2. **Создайте файл `.env` в корне проекта:**

```bash
cp .env.example .env
# Отредактируйте .env с вашими настройками
```

**Важно:** 
- Если `API_KEY` не установлен, защита API будет отключена (режим разработки)
- Если `API_KEY` установлен, он должен быть одинаковым для backend и frontend, для создания api_key можно воспользоватся ультилитой openssl: openssl rand -hex 32
- По умолчанию `VITE_API_URL=/api` (используется проксирование через nginx)

3. **Запустите все сервисы:**

```bash
docker-compose up -d
```

4. **Инициализируйте базу данных:**

```bash
chmod +x docker-init.sh
./docker-init.sh
```
5. **Откройте приложение:**

- Frontend: <http://localhost:3000>
- Backend API: <http://localhost:5000>
- PostgreSQL: `localhost:5433` (внешний порт, внутри контейнера используется стандартный 5432)

**Примечание:** Если порт 5432 уже занят локальным PostgreSQL, Docker использует порт 5433 для внешнего доступа.
**Важно:** Порты можно поменять в .Dockerfile и пересобрать контейнеры.

### Остановка

```bash
docker-compose down
```

### Остановка с удалением данных

```bash
docker-compose down -v
```

**Внимание:** Это удалит все данные из базы данных!

### Просмотр логов

```bash
# Все сервисы
docker-compose logs -f

# Конкретный сервис
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f db
```

### Пересборка контейнеров

После изменений в коде:

```bash
docker-compose up -d --build
```

## Установка

### Требования

- Node.js (v16+)
- PostgreSQL (v12+)

### Установка PostgreSQL

**Ручная установка (Ubuntu/Debian):**

```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

**Проверка установки:**

```bash
sudo systemctl status postgresql
```

### Настройка базы данных

**Создайте пользователя и базу данных PostgreSQL:**

```bash
# Переключитесь на пользователя postgres
sudo -u postgres psql

# В консоли PostgreSQL выполните:
CREATE USER postgres WITH PASSWORD 'postgres';
ALTER USER postgres CREATEDB;
CREATE DATABASE water_management OWNER postgres;
\q
```

Или одной командой:

```bash
sudo -u postgres createdb water_management
sudo -u postgres psql -c "ALTER USER postgres WITH PASSWORD 'postgres';"
```

**Настройте переменные окружения:**

**Backend (`backend/.env`):**

```bash
cp backend/env.example backend/.env
# Отредактируйте backend/.env с вашими настройками БД
```

**Frontend (`frontend/.env`):**

```bash
cp frontend/.env.example frontend/.env
# Отредактируйте frontend/.env с вашими настройками API
```

Настройки:

- При изменении пароля пользователя postgres обновите его в `backend/.env`
- `API_KEY` в backend и `VITE_API_KEY` в frontend должны совпадать
- Если `API_KEY` не установлен, защита API будет отключена

**Инициализируйте схему базы данных:**

```bash
cd backend
node config/init-db.js
```

**Выполните миграции:**

```bash
cd backend
node config/migrate-add-phone.js
```

**Создайте тестовых пользователей:**

```bash
cd backend
node config/create-test-user.js
```

Это создаст трех тестовых пользователей:

- **admin** / **admin123** (Разработчик)
- **dispatcher** / **dispatcher123** (Диспетчер)
- **plumber** / **plumber123** (Слесарь)

## Роли пользователей

- **development** (Разработчик) - полный доступ ко всем функциям системы
  - Может создавать заявки
  - Может просматривать все заявки
  - Может изменять статусы заявок
  - Может редактировать время выполнения заявок
  - Может удалять заявки и схемы на карте

- **dispatcher** (Диспетчер) - принимает и управляет заявками
  - Может создавать заявки (при клике на карте)
  - Может просматривать все заявки
  - Может изменять статусы заявок
  - Может редактировать свои заявки

- **plumber** (Слесарь) - просматривает заявки
  - Может просматривать все заявки
  - Может изменять статусы заявок, которые ему назначены

## Бригады

- **водосеть** - бригада водосети
- **канализация** - бригада канализации

## Функции системы

1. **Карта**
   - Отображение Yandex Maps
   - Визуализация схем канализации (оранжевые линии) и водопровода (синие линии)
   - Клик на карте открывает форму создания заявки (для диспетчера и разработчика)
   - Отображение существующих заявок в виде маркеров
   - Поиск адресов с предустановленным значением "Сарапул, ул. "

2. **Заявки**
   - Создание заявки с указанием адреса, описания, номера телефона, бригады
   - Просмотр списка заявок с фильтрацией по статусу, бригаде и дате (от/до)
   - Статистика заявок: количество новых, в работе, выполненных и ложных
   - Фильтрация заявок по клику на статистику
   - Изменение статусов заявок
   - Отслеживание координат заявки
   - Редактирование времени выполнения заявок (только для разработчика)

3. **Схемы на карте**
   - Сохранение схем канализации и водопровода в формате GeoJSON
   - Отображение схем на карте разными цветами
